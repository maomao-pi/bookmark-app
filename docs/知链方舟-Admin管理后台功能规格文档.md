# 知链方舟 - Admin 管理后台功能规格文档

## 一、项目背景

### 1.1 现有系统概述

「知链方舟」是一个基于 React + TypeScript + Vite + Ant Design 的网站收藏夹管理应用，目前前端功能已完整实现，数据存储在浏览器 localStorage 中（无后端服务）。

### 1.2 现有功能清单

| 模块 | 功能点 | 说明 |
|------|--------|------|
| **收藏管理** | 添加收藏 | 填写标题、网址、描述、分类 |
| | 编辑收藏 | 修改已有收藏信息 |
| | 删除收藏 | 确认后删除 |
| | 查看详情 | 弹窗展示完整信息 |
| | 复制链接 | 一键复制 URL |
| | 打开链接 | 新标签页打开 |
| **分类管理** | 创建分类 | 输入分类名称 |
| | 编辑分类 | 修改分类名称 |
| | 删除分类 | 级联处理关联收藏 |
| | 分类筛选 | 下拉选择筛选 |
| **文章关联** | 添加文章 | 关联到指定收藏 |
| | 编辑文章 | 修改文章信息 |
| | 删除文章 | 移除关联 |
| | 文章类型 | article / video / document / link |
| **搜索功能** | 实时搜索 | 匹配标题、描述、URL、分类、文章标题 |
| **主题切换** | 浅色/深色模式 | localStorage 持久化 |
| **用户认证** | 用户登录 | 登录后可添加收藏 |
| | 用户登出 | 退出登录 |
| **发现页面** | 管理员配置内容 | 用户只读浏览 |
| | 发现分类管理 | 技术博客、开发工具、文档、设计资源 |
| **排序设置** | 按日期排序 | 升序/降序 |
| | 按标题排序 | A-Z / Z-A |
| | 按来源排序 | 升序/降序 |
| **AI 功能** | URL 元数据提取 | 标题、描述、favicon、内容类型 |
| | 描述生成 | 简洁/详细/功能导向三种风格 |
| | 标签建议 | AI 推荐相关标签 |
| | 内容类型检测 | 自动识别 article/video/document |
| | API 密钥配置 | GLM-4.6 API 集成 |
| | 缓存管理 | 24 小时缓存机制 |

## 二、Admin 管理后台需求分析

### 2.1 核心目标

将现有的纯前端 localStorage 数据存储模式升级为**后端 API + 数据库**架构，并配套开发 Admin 管理后台，实现对系统数据的全面管理和运营监控。

### 2.2 用户角色

| 角色 | 权限范围 |
|------|----------|
| **超级管理员** | 完整系统管理权限，可管理所有管理员 |
| **普通管理员** | 内容管理、数据查看，无系统配置权限 |

### 2.3 技术选型（多选一）

本项目后端支持 **Python**、**Java**、**Node.js** 三种语言开发，数据库统一使用 **MySQL**。

#### 选项一：Node.js（推荐）

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 后端框架 | NestJS / Express | NestJS 提供完整 Admin 架构，Express 更轻量 |
| 数据库 | MySQL 8.0 | 持久化存储 |
| ORM | Prisma / TypeORM | TypeScript 原生支持 |
| 认证 | JWT | 管理员身份验证 |
| 前端管理后台 | React + Ant Design | 与现有前端技术栈一致 |

**优势**：
- 与前端技术栈统一（JavaScript/TypeScript 全栈）
- 团队学习成本低
- NestJS 内置 Admin 模块，快速开发
- 生态丰富，AI 集成方便

**适用场景**：快速迭代、团队熟悉 JS/TS

---

#### 选项二：Python

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 后端框架 | Django / FastAPI | Django 内置 Admin，FastAPI 更现代化 |
| 数据库 | MySQL 8.0 | 持久化存储 |
| ORM | Django ORM / SQLAlchemy | Python 主流 ORM |
| 认证 | JWT / Django Session | 管理员身份验证 |
| 前端管理后台 | React + Ant Design | 与现有前端技术栈一致 |

**优势**：
- Django 自带 Admin 后台，减少开发量
- FastAPI 现代高性能，适合微服务
- Python AI 生态丰富（PyTorch、TensorFlow）
- 开发效率高

**适用场景**：有 Python 团队、需要 Django Admin、需要 AI 深度集成

---

#### 选项三：Java

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 后端框架 | Spring Boot | Java 生态最成熟框架 |
| 数据库 | MySQL 8.0 | 持久化存储 |
| ORM | MyBatis-Plus / JPA | 简化数据库操作 |
| 认证 | JWT / Spring Security | 管理员身份验证 |
| 前端管理后台 | React + Ant Design | 与现有前端技术栈一致 |

**优势**：
- Spring Boot 生态成熟稳定
- 企业级应用首选
- 性能优秀，适合高并发
- 丰富的第三方库支持

**适用场景**：企业级应用、需要高并发、团队熟悉 Java

## 三、功能模块规划

### 3.1 登录认证模块

#### 1. 管理员登录
- **功能描述**：管理员进入后台的入口
- **功能点**：
  - 用户名/邮箱 + 密码登录
  - 图形验证码（可选，防止暴力破解）
  - 登录失败锁定机制
  - 「记住我」功能
  - 忘记密码（邮件找回）
- **数据流**：
  ```
  前端 → 登录表单 → JWT 认证 API → 返回 token → 存入 localStorage/cookie
  ```

#### 2. 权限管理
- **功能描述**：管理管理员账号和权限
- **功能点**：
  - 管理员列表展示（表格）
  - 新增管理员（仅超级管理员）
  - 编辑管理员信息（用户名、角色、状态）
  - 删除/禁用管理员
  - 角色分配（超级管理员 / 普通管理员）
  - 操作日志记录

#### 3. 个人设置
- **功能描述**：管理员个人信息管理
- **功能点**：
  - 修改密码
  - 修改头像
  - 修改联系方式

---

### 3.2 用户管理模块

#### 1. 用户列表
- **功能描述**：查看和管理注册用户
- **功能点**：
  - 用户表格展示（分页、排序、筛选）
  - 搜索用户（用户名、邮箱）
  - 用户详情查看
  - 查看用户收藏数量
  - 查看用户收藏列表
  - 禁用/启用用户
  - 删除用户（级联删除收藏）
- **数据字段**：
  - 用户 ID、用户名、邮箱、头像
  - 注册时间、最后登录时间
  - 收藏数量、状态（正常/禁用）
  - 操作（查看、编辑、禁用、删除）

#### 2. 用户详情
- **功能描述**：查看单个用户的完整信息
- **功能点**：
  - 基本信息（用户名、邮箱、头像、注册时间）
  - 收藏统计（总数、分类分布）
  - 收藏列表（可查看详细内容）
  - 登录日志

---

### 3.3 收藏管理模块

#### 1. 收藏列表
- **功能描述**：管理所有用户的收藏数据
- **功能点**：
  - 全局收藏表格（分页、排序）
  - 搜索收藏（标题、网址、描述）
  - 按分类筛选
  - 按用户筛选
  - 按来源筛选（用户收藏 / 发现内容）
  - 批量删除收藏
  - 导出收藏数据（Excel/CSV）
- **数据字段**：
  - 收藏 ID、标题、网址、描述
  - 所属用户/管理员
  - 分类、标签
  - favicon、缩略图
  - 关联文章数、创建时间、更新时间

#### 2. 收藏审核（可选）
- **功能描述**：审核用户提交的收藏
- **功能点**：
  - 待审核列表
  - 审核通过
  - 审核拒绝（可填写拒绝理由）
  - 批量审核

#### 3. 发现内容管理
- **功能描述**：管理「发现」页面的推荐内容
- **功能点**：
  - 发现内容列表
  - 新增发现内容
  - 编辑发现内容
  - 删除发现内容
  - 排序/置顶
  - 显示/隐藏

#### 4. 批量操作
- **功能描述**：对收藏进行批量处理
- **功能点**：
  - 批量移动分类
  - 批量添加/移除标签
  - 批量删除
  - 批量导出

---

### 3.4 分类管理模块

#### 1. 分类列表
- **功能描述**：管理系统分类
- **功能点**：
  - 分类树形结构展示
  - 新增分类（支持多级分类）
  - 编辑分类名称
  - 删除分类（提示关联收藏数量）
  - 分类排序
  - 显示/隐藏

#### 2. 发现分类管理
- **功能描述**：管理发现页面的分类
- **功能点**：
  - 发现分类列表
  - 新增/编辑/删除发现分类
  - 分类图标设置
  - 分类排序

---

### 3.5 文章管理模块

#### 1. 文章列表
- **功能描述**：查看所有关联的文章
- **功能点**：
  - 全局文章列表
  - 按收藏筛选
  - 按类型筛选（article/video/document/link）
  - 搜索文章（标题、网址）
  - 删除文章

---

### 3.6 系统设置模块

#### 1. AI 功能配置
- **功能描述**：管理 AI 功能开关和参数
- **功能点**：
  - GLM API 密钥配置
  - AI 功能开关（启用/禁用）
  - 自动提取元数据开关
  - 自动生成描述开关
  - 请求超时设置
  - 缓存时间设置
  - API 测试连接

#### 2. 主题配置
- **功能描述**：管理系统主题
- **功能点**：
  - 默认主题设置（浅色/深色）
  - 主题色配置
  - 允许用户切换主题（开关）

#### 3. 存储配置
- **功能描述**：配置数据存储策略
- **功能点**：
  - 数据库连接配置
  - 数据备份/恢复
  - 缓存策略配置

---

### 3.7 数据统计模块

#### 1. 概览仪表盘
- **功能描述**：系统数据总览
- **功能点**：
  - 用户总数、活跃用户数
  - 收藏总数、分类分布
  - 文章总数、类型分布
  - 今日新增收藏数
  - 近期趋势图表

#### 2. 用户统计
- **功能描述**：用户相关数据分析
- **功能点**：
  - 用户增长趋势图
  - 用户活跃度分析
  - 用户收藏排行 TOP N

#### 3. 收藏统计
- **功能描述**：收藏数据分析
- **功能点**：
  - 分类收藏数量分布
  - 热门网站排行
  - 标签使用频率统计

---

### 3.8 操作日志模块

#### 1. 日志列表
- **功能描述**：记录管理员操作
- **功能点**：
  - 操作日志列表
  - 按管理员筛选
  - 按时间范围筛选
  - 按操作类型筛选
  - 日志详情查看

#### 2. 日志字段
- 操作时间
- 管理员账号
- 操作类型（新增/编辑/删除/登录等）
- 操作对象（用户ID/收藏ID等）
- IP 地址
- 操作结果（成功/失败）

## 八、注意事项

1. **数据迁移**：现有 localStorage 数据需设计迁移脚本
